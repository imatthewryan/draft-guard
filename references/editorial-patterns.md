# Editorial patterns

Use these as evidence-based checks. A pattern is not proof of AI authorship. Keep it when it serves the writer's voice, genre, or meaning.

| Pattern | Signal | Better move | Keep when |
|---|---|---|---|
| Throat clearing | “Here's the thing” or a generic opening | State the point | The opener adds a real story, stake, or voice |
| Empty abstraction | “Improves efficiency” without a mechanism | Name the action, result, or number | The source cannot support more detail |
| Importance puffery | “A pivotal moment” or “a testament” | State the concrete fact | The phrase is part of a deliberate voice |
| Weasel attribution | “Experts agree” without a source | Name the source or remove the claim | The source is named nearby |
| Fake contrast | “It is not X. It is Y.” | State Y directly | Contrast itself matters |
| Rhetorical setup | “What if I told you?” or “Think about it” | Make the claim | The format is intentionally conversational |
| Faux-insight setup | “What nobody tells you” | Remove the status claim | The line is a quoted subject of analysis |
| Forced symmetry | Repeated sentence shapes or lists of three | Use the number and shape the content needs | Rhythm is a clear stylistic choice |
| Synonym cycling | One thing gets three names | Repeat the clearest name | The terms are truly different |
| Decorative verb | “Serves as a hub” or “drives transformation” | Name what it does | The verb is literal and specific |
| Content-marketing metaphor | “Unlock,” “landscape,” “ecosystem,” “deep dive” | Name the action, place, or system | The metaphor carries real meaning |
| Summary ending | “In conclusion” or a restated final paragraph | End on the last concrete point | The format needs a formal recap |
| Decorative fragmentation | Stacked fragments used as a fake punchline | Join or cut them | Fragmentation is part of the speaker's voice |
| Decorative em dash | Dashes used as default rhythm | Use a period, comma, or parentheses | The dash clearly improves the sentence |
| Empty hedge | “Powerful, but not without drawbacks” | State the actual tradeoff | Specific limits follow |
| Over-formatting | Tiny sections, decorative bold, needless bullets | Match form to content | The format aids scanning or access |

## Hard constraints

Never apply a pattern pass to protected code, commands, URLs, citations, quoted text, data, or user-specified terms. Never use a pattern label as an authorship verdict.
